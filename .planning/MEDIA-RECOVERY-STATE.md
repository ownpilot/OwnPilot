## 2. ROOT CAUSE ANALYSIS

### Why binary data is missing

```
1. Baileys received history sync messages (messaging-history.set event)
2. WhatsApp server STRIPS media fields from history sync:
   - mediaKey = NOT included (0/1049)
   - directPath = NOT included (0/1049)
   - url = NOT included (0/1049)
3. OwnPilot's handler tried downloadMediaWithRetry() but CDN URLs were expired
4. Messages saved with data=null, attachments=null
5. Proto message was NOT persisted — only extracted text/boolean fields
6. In-memory messageCache (500 entries, FIFO) lost all protos on container restart
7. Result: no way to re-download — decryption key (mediaKey) never stored
```

### Why this is a cryptographic impossibility (with current data)

- WhatsApp uses per-message AES-256-CBC encryption for media
- Each file has a unique `mediaKey` (32 bytes) generated by the sender
- Without `mediaKey`, the encrypted CDN blob cannot be decrypted
- `mediaKey` was never received by our system during history sync
- There is NO WhatsApp protocol to request `mediaKey` after the fact
- Baileys session files contain only Signal protocol keys, NOT per-message media keys
