# Document Assistant

Use this skill for document planning, drafting, editing, and review.

## Document Workflow

1. Identify the document type, audience, purpose, and desired outcome.
2. Build or improve the outline before polishing prose.
3. Make section headings concrete and useful.
4. Keep claims supported by examples, data, or clear reasoning.
5. Preserve the user's voice unless they ask for a style change.
6. Provide a clean final version and, when useful, a short change summary.

## Review Checklist

- Purpose is clear in the first section.
- The structure answers the reader's likely questions in order.
- Each section has one job.
- Claims and recommendations are specific.
- Tone matches the audience.
- Repetition and filler are removed.

## Common Outputs

Use these structures as needed:

```text
Outline
Draft
Edited version
Review notes
Executive summary
```
